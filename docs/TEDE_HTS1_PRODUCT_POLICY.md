## 2. 분양(화이트라벨) — 거래소·고객사별

**목적:** 서로 다른 거래소·고객이 **같은 회사의 동일 제품**으로 인식하지 않도록, 겉모습만 차별화.

### 허용 (스킨·설정)

- 색상·타이포·간격 (CSS 변수, `data-theme`, 테마 프리셋)
- 패널 **기본 배치** (도킹 레이아웃 preset)
- 로고·회사명·상태바·타이틀 문구
- 기본 워치리스트·심볼 탭 템플릿 (설정 JSON)

### 금지 (분양 빌드에서)

- TP/SL·헷지·원웨이·MIT/STOP 등 **주문·포지션 규칙** 변경
- 테마(Shell)마다 `tradingSession` / `orderFlow` 로직 **복사·분기**
- “이 거래소만” 기능을 23에 추가

### 구현 원칙

- **엔진·워크플로** (`src/engine/`, `useTradingWorkflow`) — 테마 무관, **한 곳**
- **테마** (`ThemeRootShell`, `theme1` / `theme2`, 향후 `themeN`) — **조립·스타일만**
- 분양 = 빌드 플래그 + `theme*Presets` + 브랜드 에셋. 동작은 테데HTS1과 **100% 동일**해야 함.

## 3. 기능 동결 (FREEZE)

### 동결 선언 조건

- `23_DEV_SCHEDULE.md` **P1~P2** Gate A/B 통과
- `23_VERIFICATION_REPORTS.md`에 동결 보고 (예: `VR-FREEZE-TEDE-HTS1`)
- Git 태그 후보: `STABLE_TEDE_HTS1_V1` (또는 스케줄의 `STABLE_23_COIN_MOCK_V1`와 함께 명시)

### 동결 후 23에서 허용

| 유형 | 예 |
|------|-----|
| 버그 수정 | 주문 미체결·표시 오류·회귀 |
| 분양 스킨 | §2 허용 항목 |
| 문서·QA | 체크리스트·수동 QA 기록 |
| Gate 유지 | lint / build / test / smoke |

### 동결 후 23에서 **금지**

- 신규 주문 유형·새 상품 규칙·실 API·02-TGX-CEX 연동
- “분양 A 요청”으로 들어온 **신기능**
- 테마별 기능 드리프트 (한 테마에만 있는 동작)

## 5. 에이전트·개발자 작업 규칙

1. **Gate 실패 시 즉시 중단** — `23_DEV_SCHEDULE.md` Phase 순서만 진행.
2. 분양 요청이 **색·위치·브랜드**면 23 설정/테마 레이어.
3. 분양 요청이 **기능**이면 **24로 안내** — 23에 구현하지 않음.
4. UI만 변경해도 **Gate A** 필수.
5. 테마 추가 시 **공통 패널·훅 재사용** — Shell에 비즈니스 로직 넣지 않음.
